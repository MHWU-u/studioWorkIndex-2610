Featured Studio Project

Emoji Landscape Prototype

Emoji Landscape is a browser-based creative tool that allows users to build small symbolic landscapes and city scenes using emoji. Instead of drawing with brushes or lines, users create compositions by selecting, placing, rotating, scaling and mirroring visual symbols on a 9×9 grid.

The project explores emoji as a shared digital visual language. It focuses on mapping and feedback: each control immediately changes how the emoji appears, helping users understand the relationship between interaction and visual result.

1. Conceptual response to the project brief

This project responds to the brief by creating a small browser-based creative tool that lets users make images with emoji. Instead of drawing with lines or brushes, the user builds a composition by placing and changing visual symbols.

The idea comes from thinking about emoji as a shared digital language. They are usually used in messages, but they become a material for image-making in this project. A key reference for this project is Xu Bing’s Book from the Ground, which uses symbols to communicate across language. And it helped me think about how emoji can work as a simple and accessible visual system.

The project mainly explores mapping and feedback. Mapping shows in how the controls affect each emoji, such as scale and rotation. Feedback shows in the immediate visual change on screen. Overall, the project is meant to be a small expressive tool that is easy to use, playful, and open to experimentation.

2. Technical approach to implementation

The main features are generating emoji, placing them on the stage, and changing their appearance. Users can adjust scale, rotation, and mirror effects to build different kinds of compositions. JavaScript stores and updates these changes, while CSS is used to display them visually.

I want to keep the implementation simple and focused on the main idea of the project. The goal is to create a small creative tool with a clear interaction system. Code comments in the prototype will also explain how the design choices connect to the concept.

5. Reflecting on responses from peer user testing

The peer user testing showed that the prototype is already working well as a simple creative tool. Most users were able to make an image they liked, and many used it through exploration. This suggests that the project is successful in encouraging playful experimentation, which is one of its main aims. The controls were also generally easy to understand, and the random emoji feature was seen as helpful for creativity.

At the same time, the feedback also showed some areas to improve. A few users suggested making the layout clearer so the controls and grid can be seen more easily together. Others mentioned that features such as mirror could be more obvious, and that resetting scale and rotation when a new emoji is generated might make the interaction smoother.

Some users also wanted a little more control, such as an emoji selector, a delete function, or a download option. Overall, the testing confirmed that the prototype has a strong base, and the next step is to refine the interface and add a few small improvements.

6. Project development timeline

So far, the project has developed from a broad idea about visual communication into a more focused emoji-based drawing tool. Early experiments helped me test interaction ideas such as random generation, placement, and transformation.

Compared with the initial plan, I’m planning to make the project more diverse, create a more interesting user experience and encourage users' creativity.

In the next four weeks, I plan to improve the prototype, respond to peer testing feedback, refine the interface, and prepare the final version for submission.
