1. Conceptual response to the project brief

This project responds to the brief by creating a small browser-based creative tool that lets users make images with emoji. Instead of drawing with lines or brushes, the user builds a composition by placing and changing visual symbols.

The idea comes from thinking about emoji as a shared digital language. They are usually used in messages, but here they become a material for image-making. A key reference for this project is Xu Bing’s Book from the Ground, which uses symbols to communicate across language. That helped me think about how emoji can work as a simple and accessible visual system.

The project mainly explores mapping and feedback. Mapping appears in how the controls affect each emoji, such as scale, rotation, and mirroring. Feedback appears in the immediate visual change on screen. Overall, the project is meant to be a small expressive tool that is easy to use, playful, and open to experimentation.

2. Technical approach to implementation

The project will be built with HTML, CSS, and JavaScript. HTML provides the basic page structure, CSS controls the layout and visual style, and JavaScript handles the interactive parts of the tool.

The main features are generating emoji, placing them on the stage, and changing their appearance. Users can adjust scale, rotation, and mirror effects to build different kinds of compositions. JavaScript stores and updates these changes, while CSS is used to display them visually.

I want to keep the implementation simple and focused on the main idea of the project. The goal is not to make a complex drawing app, but to create a small creative tool with a clear interaction system. Code comments in the prototype will also explain how the design choices connect to the concept.

6. Up-to-date project development timeline

So far, the project has developed from a broad idea about visual communication into a more focused emoji-based drawing tool. Early experiments helped me test interaction ideas such as random generation, placement, and transformation. Because of this, the project became simpler and more manageable.

Compared with the initial plan, I decided to reduce the number of features and focus more on the core interaction. This makes the project easier to complete and keeps it closer to the brief.

In the next four weeks, I plan to improve the prototype, respond to peer testing feedback, refine the interface, and prepare the final version for submission.
