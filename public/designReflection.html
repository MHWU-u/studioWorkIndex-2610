Emoji Landscape

A reflection on the process of designing my final studio project.

1. Project Purpose and Initial Direction

For my final studio project, I designed Emoji Landscape, a small browser-based creative tool for making symbolic scenes with emoji. Instead of drawing with lines, brushes or complex illustration software, users can choose an emoji, transform it and place it onto a 9×9 grid. They can build a landscape, a city scene or a more abstract composition through a series of simple decisions.

My initial judgement of purpose was to make image-making feel more accessible and less intimidating. I wanted to create a tool that did not require advanced drawing skills or a long learning process. Emoji already exist in everyday digital communication, so most users understand them without needing further explanation.

The final design still follows this original purpose. The tool is simple enough for a user to understand quickly, but it does not only produce one type of image. Users can generate a random emoji or type their own symbol. They can change its scale, rotation and direction before placing it. They can also revise individual cells and download their final composition. These functions allow users to make personal choices without making the interface too complicated.

2. Judgement of Worth

At the beginning of the project, I identified several values that I wanted the tool to communicate: accessibility, playfulness, learnability and open-endedness. These values guided my design decisions throughout the process.

Accessibility is important. The grid provides a clear starting point. Emoji are also familiar symbols, so users do not need to learn a new visual language before they begin.

Playfulness is embodied through the random emoji function. Users may receive a symbol that they did not expect, and this can influence the direction of the composition. For example, a user may start with a simple landscape but then receive a car, a building or an animal.

Learnability is supported by the limited number of controls. Users can see what happens when they change the scale, rotate an emoji or switch on mirror mode.

Finally, the tool is open-ended. A composition can be a recognisable landscape, but it can also be a pattern, a symbolic scene or a collection of unrelated objects.

3. Interaction Principles: Mapping and Feedback

The two main interaction design principles in my project are mapping and feedback.

Mapping refers to the relationship between a user action and the result that appears on screen. In Emoji Landscape, each control has a visible and predictable effect.

Feedback is also immediate. The current preview updates before the emoji is placed, and the grid responds directly after the user clicks. This helps users understand the system through experimentation.

4. Design Judgements Through Exploration

• Keeping the 9×9 Grid

One important design judgement was whether I should keep the grid or change the tool into a fully free canvas. A free canvas would give users more spatial freedom. They could place symbols anywhere and make more detailed compositions.

After considering the purpose of the project, I decided to keep the 9×9 grid. The limitation is not only a technical shortcut. It is part of the creative method. The grid reduces hesitation because users do not need to think about exact coordinates or small movements.

I think this limitation is appropriate for a small creative tool. It gives the project a clearer identity and prevents it from becoming a simplified version of a conventional drawing application.

• Adding Local Editing

Peer feedback showed that users wanted an easier way to correct small mistakes. In the earlier version, clearing the entire grid was too extreme when the user only wanted to remove one emoji. For this reason, I added a single-cell erase mode.

This is a small function, but it changes the experience of using the tool. Users can revise their image gradually instead of starting again.

5. Finalised Production Timeline

My project developed through several stages:

Stage Main Activity Reflection
Early studio experiments I explored information design, feedback design and mapping through smaller browser-based tasks. These experiments helped me understand that a simple interaction can still create an expressive result.
Concept development I decided to use emoji as image-making materials. This gave the project a clear visual identity and connected it with everyday digital culture.
Prototype development I created the first grid-based version with generate emoji, transformation controls and placement functions. The prototype validated the concept, but its functionality was too simple to showcase the user's creativity.
Expanded prototype iteration I developed a more complex version of the prototype. I added different emoji categories, a larger emoji selection system, mirror mode and canvas themes. This stage was useful because it helped me explore how far the tool could be expanded. However, it also made the interface more crowded.
Peer user testing Users tested the tool and suggested manual emoji input, easier deletion and image download. The feedback helped me see that users wanted more choices, but they also wanted the interface to remain easy to understand.
Final refinement I simplified emoji selection, added manual input, single-cell erase mode and download functionality, and kept the 9×9 grid. The final version is more focused and more usable.

The production process was not completely linear. Some ideas that seemed useful at first were removed later. For example, the emoji palette appeared to offer more choice, but it also created more visual clutter. I learned that removing features can be as important as adding them.

6. What I Am Most Proud Of

The aspect I am most proud of is the balance between limitation and user agency. The tool is deliberately small in scope, but users still have different ways to make the composition their own. They can choose between random and manual input, adjust the appearance of symbols, revise individual cells and export the image.

I also think the project communicates its character clearly. It is not trying to become professional illustration software. It is a playful symbolic stage.

7. The Most Challenging Aspect

The most challenging aspect was deciding how many functions the tool should include. During the process, it was tempting to add more features, such as more emoji categories, drag-and-drop placement or a completely free canvas. These ideas might create more possibilities, but they could also make the tool harder to learn.

I needed to think carefully about the purpose of each function. I tried to ask whether a new feature supported the main experience or only made the tool look more complex. This process taught me that interaction design is not only about solving technical problems. It is also about making clear decisions and accepting that some limitations are useful.

8. Learning from a Peer Creative Tool

• Klotz Beats

https://klotzbeats.netlify.app/

One peer creative tool that provided an insightful contrast to my own design judgement was Klotz Beats. In this project, users place blocks inside a grid to build a short musical composition. The interaction feels similar to using a simple beat pad or sequencer. It is easy to understand because users can quickly see how different blocks are arranged, make small adjustments and listen to the result.

This project gave me a useful point of comparison because both Klotz Beats and Emoji Landscape use a grid as the main structure. Testing this tool helped me reflect on my decision to keep a fixed grid rather than move towards a fully free canvas. A grid can sometimes appear restrictive, but Klotz Beats shows that limitation can also make creative interaction more immediate. Users do not need professional music-making experience before they begin. They can learn through placing blocks, listening and adjusting the pattern.

9. General Feedback on Peer Creative Tools

• Peer Project Feedback 1: Breath Visualization

https://breathtrack1.netlify.app/

Breath Visualization creates a calm and immersive interaction experience through a very minimal visual system. When I tested the tool, I was especially interested in how the expanding circles respond to the breathing process. Together with the background music, the movement feels slow and relaxing. The interaction is simple, but it creates a clear atmosphere.

One strength of this project is that the visual design and the purpose of the tool are closely connected. The interface does not include too many distracting elements. This makes sense because the user is encouraged to slow down and focus on a repeated action. The expanding rings are also an effective form of visual feedback. They make an invisible action, breathing, feel more visible and tangible.

• Peer Project Feedback 2: Ripple

https://ripple-studioproject.netlify.app/

Ripple was one of the most playful projects that I tested. Users can place different sea creatures into different pitch areas and listen to the sounds created by their movement and collisions. I found the experience interesting because the user can influence the system, but cannot completely control the final result. The creatures move in their own ways, so the composition continues to change after they are placed.

The visual style is also very suitable for the interaction. The sea creatures, water ripples and soft colour palette create a light and friendly atmosphere. This makes the tool feel approachable, even though the sound system behind it is more complex. I particularly liked the way the project uses different creatures as more than decorative images. Each creature has a different movement behaviour, while stones can redirect the collision paths. This gives users reasons to experiment with different combinations.

One possible improvement would be to make the musical differences between the creatures more visible before users place them into the pool. The hover descriptions are useful, but a small visual cue or short preview sound could help users understand the available choices more quickly.

• Peer Project Feedback 3: Distortion Playground

https://s4158360distortionplaygroundfinal.netlify.app/

Distortion Playground uses a simple dot grid to create a surprisingly dynamic interaction experience. Users click and drag across the canvas to distort the dots, and the movement changes depending on the selected mode. I enjoyed testing the difference between Liquid, Elastic and Heat because each mode gives the same basic input a different physical feeling.

The strongest part of the project is its immediate feedback. The interface is visually minimal, but the response feels lively and complex. A small dragging gesture can create a much larger movement across the canvas. This makes the tool easy to start using, while still allowing users to spend time experimenting with different effects.

I also think the project uses visual simplicity effectively. The basic dot element does not distract from the interaction itself. Instead, it allows the user to pay attention to movement, density, attraction and distortion. The additional controls give users more ways to shape the result, but the main interaction remains clear.
