Working title: “Emoji Drawing as Shared Symbolic Composition”

A browser-based interactive drawing page that lets users compose images through random emoji rather than freehand marks.

Purpose

This project proposes an interactive web page for drawing with emoji. Instead of asking the user to draw lines, paint shapes, or demonstrate traditional illustration skills, the page provides a sequence of ready-made emoji that can be placed, scaled, rotated, and mirrored across the composition. The user builds an image by selecting where each symbol should go and how it should behave visually. In this sense, the project shifts drawing away from manual mark-making and toward arrangement, relation, and symbolic construction.

The project is inspired by Xu Bing’s Book from the Ground, which constructs communication through symbols, pictograms, and signs that can be broadly recognised without relying on one spoken language. That precedent is useful for my project because emoji already operate as an informal shared visual vocabulary on the internet. They are not neutral images: each one carries cultural associations, moods, gestures, and references. By turning emoji into the basic material of a drawing tool, I want to explore whether users can build images through existing symbolic forms instead of starting from a blank drawing system.

The purpose of the project is therefore not only to make a playful tool, but also to test a different idea of what digital drawing can be. Rather than rewarding technical drawing ability, the page encourages composition through selection, positioning, repetition, distortion, and juxtaposition. It asks whether an image can emerge from shared symbols and whether visual expression can become more accessible when the user is not required to “draw well” in a conventional sense.

Worth

I think this project is worth making because it lowers the threshold for participation while still producing a meaningful form of visual authorship. Many drawing interfaces are built around precision, skill, and confidence: the user is expected to control the cursor accurately and translate an idea into a line. My project takes a different position. It offers a set of already-formed visual units and asks the user to organise them into a scene. This makes the tool more approachable for people who may feel excluded by conventional drawing software, while still leaving room for interpretation, humour, mood, and composition.

The project is also worth pursuing because emoji are already part of everyday digital communication, but they are usually used within text rather than treated as the primary material of image-making. Reframing emoji as movable, transformable visual objects allows the project to question the boundary between writing, interface symbols, and drawing. A crying face, a tree, a flame, or a shoe can become less like “text decoration” and more like a modular pictorial element. This gives the work a conceptual dimension: it is not simply a novelty website, but a small experiment in visual language.

Another reason the project has value is that randomness can create productive surprise. Because the emoji appear through random selection, the user does not begin with total control. Instead, composition emerges through negotiation between choice and chance. This can lead to more unexpected images than a standard palette would, and it encourages users to respond creatively to what the interface gives them. In that sense, the project is not only about expression but also about interaction design: the interface becomes a collaborator that proposes material rather than simply obeying commands.

Finally, the project has worth as a web-based creative tool because it can remain simple while still feeling distinctive. It does not need to become a full illustration program. Its value lies in doing one thing clearly: letting users construct images from a shared symbolic vocabulary through direct manipulation and playful constraint.

Framing

I am framing this project as a small interactive expressive tool rather than a polished professional design application. Its main focus is not efficiency or productivity, but a specific interaction idea: composing images through pre-existing symbols. The project will therefore be deliberately limited in scope. The core interaction loop is: generate an emoji, place it on the page, transform it, and repeat. The primary features will be random emoji generation, drag-and-drop placement, scale adjustment, rotation, horizontal mirroring, and basic reset or delete functions.

This framing is important because it defines the project as an exploration of constraint. Users are free to build many different kinds of compositions, but they can only do so through a restricted symbolic system. That tension between openness and limitation is central to the concept. The page should feel playful and accessible, but it should also make users aware that they are composing with signs that already have meaning. In other words, the interface is not presenting emoji as neutral shapes; it is presenting them as cultural units that can be rearranged into new visual statements.

From an interaction design perspective, I want the framing to balance immediacy and intentionality. The tool should be easy to begin using with almost no instruction, but once the user starts placing emoji, the page should reveal more compositional possibilities through scale, rotation, mirroring, and layering. This gradual deepening is important to the experience. The project should first feel light and intuitive, then slowly become a system for experimenting with visual rhythm, orientation, density, and symbolic contrast.

I also frame the project as web-native. It should work directly in the browser with HTML, CSS, and JavaScript, and it should preserve the casual immediacy of online creative tools. This matters conceptually because emoji are themselves born from networked communication environments. Presenting them inside a browser-based composition tool keeps the project close to the digital context from which its visual language emerges.

Interactive experiments

My initial interactive experiments for this project focus on how constraint can still produce expressive freedom. I plan to test this through both the provided examples and independently researched tools that can support the project’s technical and conceptual direction.

Provided example 1: Broider

Broider is useful because it shows how a creative tool can produce strong visual variety from a very limited system. The user works with a small, rule-based structure and receives immediate feedback as the design develops. What I want to take from Broider is not its border format, but its logic of composition through fixed units. In my own project, emoji will function as those units. Rather than offering a blank drawing brush, the interface offers discrete visual components that can be combined into larger arrangements. This supports my aim of making image-making feel accessible without making it feel empty.

Broider also demonstrates that a simple interface can still feel satisfying if the response is immediate and the constraints are clear. For my project, this suggests that each emoji placement should feel visually decisive. The user should understand that every object added to the page contributes to an accumulating composition rather than disappearing into an overly complex interface.

Provided example 2: OrbiDraw

OrbiDraw is especially relevant because it frames drawing through a relational rule: the user chooses a point, and placed stamps orient relative to that point. This is important for my project because I do not want emoji placement to become only random collage. I want to test whether a simple compositional rule can generate stronger visual coherence.

Inspired by OrbiDraw, one experiment for my project is to introduce an optional “anchor mode” in which newly placed emoji rotate in relation to a central point or to the cursor position. This could make the page feel more like a system than a pile of stickers. It would also allow users to discover patterns such as spirals, radiating clusters, circular arrangements, or mirrored constellations. This experiment is useful because it tests how much structure the tool needs in order to remain playful while still producing visually interesting outcomes.

Independently researched tool 1: interact.js

For implementation, I researched interact.js because it provides browser-based drag and drop, resizing, and gesture interactions. This makes it highly relevant to my project’s central actions: moving emoji, adjusting their size, and potentially supporting touch-based interaction. Using a lightweight interaction library could help me build direct manipulation in a clearer and more stable way than writing every movement behaviour from scratch.

More importantly, interact.js supports the project conceptually because it keeps the interface focused on physical manipulation. The user should feel that each emoji is an object that can be picked up, positioned, and adjusted. That object-based behaviour matches the idea that the project is a form of composition through symbolic units rather than brush strokes.

Independently researched tool 2: html2canvas

I also researched html2canvas as a simple way to let users export their finished composition. Since the project is about making images from emoji arrangements, it is important that the user can keep or share the result. html2canvas can render a selected part of the page in the browser and turn it into an image output, which makes it a practical option for a final “save artwork” function.

This matters because it allows the website to produce an artifact rather than only a temporary interaction. The project then becomes not just a playful experiment but a tool that generates a finished visual result. That supports the project’s overall value as a small creative platform rather than a one-time interface joke.

Planned prototype direction

Based on these references, my prototype will begin with a deliberately simple version: one random emoji appears at a time, the user places it onto a composition area, and then adjusts its size, rotation, or mirrored state. Once this core loop works, I can test whether an optional orientation rule, inspired by OrbiDraw, improves the visual quality of the compositions. If the basic interaction feels stable, I will then add an export function so the final composition can be saved as an image.

In this way, the interactive experiments are not separate from the final project. They directly shape the project by helping me test which balance of freedom, randomness, transformation, and rule-based structure produces the strongest experience.
